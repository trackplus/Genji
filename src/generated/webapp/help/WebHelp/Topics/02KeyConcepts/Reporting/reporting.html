Key concepts

Queries, filters, views, and reports

To retrieve a set of items you use queries and reports. Both of them use filters.

To retrieve a set of items you use queries and reports. Both of them are based on filters.

  • Queries are executed to show a set of items in the item navigator. A query is associated with a filter and an optional view. Query filters are limited to filter on actual item property values.

  • Filters represent criteria to define the items you will see in the item navigator, a cockpit tile or a report. Filters work on actual item property values.

  • Views define how the items of a query are presented in the item navigator. You can have a Gantt view, an agile board, a flat list, etc. For each view you can define what properties to show, how to group and sort, and so on.

  • Reports are being used to create any conceivable kind of representation of a set of items. Reports are based on filters like queries, but reports can also employ their own filter engine. Thus reports are not limited to filters that filter on actual item properties but they can filter on any information in the Track+ Service Desk database. Report output formats can be PDF, Word, Excel, HTML, and others.

  • The item navigator
    To work with a set of items you use the item navigator. Next to the cockpit it is the central location from where you do most of your work.
  • Filters
    Filters are used by queries and reports to define a set of items based on a set of criteria.
  • Reports
    With reports you can obtain any information from the Track+ Service Desk database in almost any conceivable format, like graphics, PDF, Word, HTML, XML, or Excel.
