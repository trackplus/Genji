Gantt chart

The Gantt chart permits you to graphically see and modify start and end dates, work progress, and dependencies between items, and highlight the critical path.

General behaviour

The Gantt view permits you to view the items selected by a filter with their position in the work breakdown structure, their position in time, and their dependencies.

If your license permits you can also change items and dependencies directly in this view by dragging start and end dates and connecting items with dependency links.



You can find more information on underlying concepts at Bottom-up and top-down and Planning with baselines. Information on how you can enable bottom-up and top-down handling you can find at Various configuration items.

Modifying column width

  1. To modify the width of a column move the cursor to the right double bar of that column header.


  2. Select and move the bar to increase or reduce the width of the column.

Moving columns

To move a column to another place select that column header and drag it to the left separation bar of another column header.


Adding and removing columns

CAUTION:
You should not remove the item number column and the set operation column for the simple, detailed, work breakdown structure, and Gantt view mode since it enables you to drag items in the hierarchy and to perform mass operations. Remove these columns only if you do not need this functionality.

The following procedure pertains to the text-only view modes and the Gantt diagram. The agile boards are configured differently.

  1. To add or remove attributes, click on the „Choose columns“ button at the top right position of the item navigator screen. A list with available and selected attributes pops up.


  2. Mark an unchecked attribute to add it to the item navigator. Mark a checked column to remove it from the item navigator.

Folding and unfolding item hierarchy

To unfold child items, click on the „+“ button. To fold them back together click on the „-“ button in that column. To fold or unfold all items, click on the appropriate buttons in that columns header.


Moving items in the hierarchy

You can move items to another parent by dragging the item number to a new parent number, and dropping it there.

Edit modes standard, with baseline, and baseline only

When you have enabled bottom-up and top-down handling on your installation and have forms that contain both, top-down and bottom-up dates you can display the top-down values as baseline. In the configuration menu you can between showing just the current plan, showing the current plan and the baseline, and showing the baseline only. In the last mode you can edit the baseline dates directly in the view.

  1. Enable the Gantt configuration menu (the little wrench in the top right corner).
  2. Select "Show baseline" or "Show both" in the tool bar.


Changing start and end date

The Gantt diagram permits you to intuitively edit start and end dates.

Important: When you have enabled top-down support, you will see in your Gantt diagram the bottom-up values by default. You can display the top-down values via the "show baseline" function, but you can manipulate only the bottom-up values. To change the baseline values you have to switch the view to show just the baseline values.
  1. Click on the beginning or end of the bar representing the item you want to change, depending on whether you want to move the start or end date.
  2. Drag the point to the desired date and release.
  3. In case of task dependencies there may be conflicts. You will be prompted for input in case the system cannot resolve the conflicts itself.

Creating and modifying task dependencies

You can create and modify task dependencies directly in the Gantt diagram to define Finish to Start, Finish to Finish, Start to Finish, and Start to Start relationships between tasks.

  1. Open the Gantt chart configuration
  2. Drag the cursor to the successor task start or finish.
  3. Release the mouse button and enter all necessary information into the upcoming popup window.
  4. To modify an existing dependency right click on the link in the Gantt diagram. The context menu offers you a number of choices.

Showing baseline values (top-down)

When you have enabled bottom-up and top-down handling on your installation and have forms that contain both, top-down and bottom-up dates you can display the top-down values as baseline. The bottom-up values are shown only by default. When you switch to showing the baseline values only you can edit the baseline values and links directly in the diagram.

  1. Enable the Gantt configuration menu (the little wrench in the top right corner).
  2. Select "Show baseline" in the tool bar.

Validating links

You can validate links on plausibility, like end-to-start relations where the start date must be later than the end date.

  1. Enable the Gantt configuration menu (the little wrench in the top right corner).
  2. Select "Validate relationships".

Displaying the critical path

You can highlight the critical path in the Gantt diagram. The critical path is defined by the path from the latest end date to the earliest start date. A path is made up of links between items.

  1. Enable the Gantt configuration menu (the little wrench in the top right corner).
  2. Select "Validate relationships".

Task dependencies conflict resolution

When you move the start or finish date of an item, the system tries to keep the integrity of the schedule by moving dependent item dates when necessary. This can lead to conflicts.

When there are temporal task dependencies, moving the start or finish date of an item can require to move dependent items dates as well to maintain the integrity of the schedule. We consider X cases:
  1. The item being moved does not have any successors and predecessors.
  2. The item has at least one successor but does not have any predecessors.
  3. The item does not have any successor but does have at least one predecessor.
  4. The item has at least one successor and at least one predecessor.

Case 1 is trivial, there is nothing to do in terms of calculating new start and end dates based on task dependencies.

Case 2 requires to adjust the successors start and end date recursively in case of Finish to Start and Finish to Finish.
