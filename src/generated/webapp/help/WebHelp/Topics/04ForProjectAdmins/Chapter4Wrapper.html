Track+ Service Desk for space administrators

  • Managing spaces
    Track+ Service Desk organizes all items in workspaces or spaces for short. Spaces are administered by users that have space administrator privileges for their spaces. This chapter describes tasks and concepts typical for project or space administrators.
  • Assigning roles to team members
    Before anybody can do anything in a workspace or project they have to be assigned at least one role in that workspace. With each role comes a set of permissions, like being allowed to change items, to create new item, or to modify specific item properties.
  • Managing releases (phases)
    Track+ Service Desk organizes all items in workspaces. You can structure workspaces along the time line in phases, for example to manage releases, iterations or sprints. Phases can be structured hierarchically just like items. For example, you could have a release with a number of iterations or sprints as sub-phases.
  • Managing workspace specific lists
    In Track+ Service Desk you can configure any number of custom lists. You can make these lists available to all workspaces, or just to a specific workspace. Items that contain a custom list property will have one of the preconfigured values of the lists you define. For example, you could have lists with operating systems, drawing numbers, serial numbers, or customers. You could have a list with system components, or just tag items with the location where you need to take care of them.
  • Assigning accounts
    You can plan and monitor work and expenses via accounts that you assign to workspaces. Using accounts you can differentiate between different types of work and expenses, like development, administration, customer requests, and so on.
  • Default automail assignments
    For occasional users, the configuration of automail settings may require too much knowledge and time. It is therefore possible that a space administrator, system manager or system administrator configures automail settings for a workspace, or all workspaces for all users that do not overwrite these settings with their own.
  • Configuring workspace-specific cockpits
    Track+ Service Desk supports to basic perspectives: a user perspective and a workspace perspective. The user is interested in all items that concern him, possibly across a number of workspaces. A workspace or project administrator is more interested in items that are contained in his workspace or project. To facilitate cooperation you can provide workspace-specific cockpits to your team members. This helps to ensure that everybody has the same view on a workspace in discussions.
  • Version control
    With Track+ Service Desk you can link files in a version control repository like Subversion, GIT or CVS to items.
  • Generating release notes
    You can generate release notes and copy them to your personal website or include them in a document.
