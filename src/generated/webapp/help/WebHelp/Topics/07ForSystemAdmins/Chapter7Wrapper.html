Administering the server

  • Concepts and tasks for system administrators
    Some operations like connecting a Track+ server to an e-mail server, providing disk space, or performing database backups and restores are reserved to Track+ system administrators. Track+ comes with a built in special user, named „admin“, initial password „tissi“.
  • Server configuration
    There are a number of parameters that need to be configured before Track+ can run in a meaningful way. This section describes these basic system settings.
  • Cluster operation
    Track+ can be operated in a clustered environment, i.e. one Track+ instance can run on a number of servers. This section gives examples of possible configurations and explains how Track+ behaves in a clustered environment.
  • Configuring login page text
    The login page of the Track+ system provides an area where messages to existing or prospective users of the system can be placed, like downtime schedules, or registration policies. The message can be provided in several languages at the same time. This section describes how to edit this text.
  • Observing and managing server state
    Sometimes it is necessary to prevent users from accessing the system, for example if a new version is to be deployed, or other maintenance tasks have been scheduled. Server accessibility may also be effected by the number of users registered, because of license restrictions.
  • Broadcasting e-mails to Track+ users
    The system administrator can send broadcast e-mails to all or some Track+ system users.
  • Configuring diagnostic messages
    Track+ can be operated with many applications servers, database systems, and operating systems. Furthermore, Track+ can be configured in numerous ways. Should there be any problems in such configurations, it is very helpful for the support team to get diagnostic messages.
  • Data backup
    A system administrator can backup all data from the database and optionally all attachments into a ZIP archive file. Furthermore it is possible to create such backups automatically on a regular basis. The backup data can be used to restore the database and all attachments at a later point in time.
  • Restoring data from backup
    A system administrator can restore data from a backup file to recover from a crash, port data to a new system, or go back to previous date.
  • Switching database systems
    It is possible to switch the Track+ database from one database system to another one. For example, this may happen when you started out using the Track+ Windows installer that comes with the Firebird RDBMS, and later on want to move to an RDBMS officially supported by your company.
  • Non-western character sets
    Internally Track+ uses Unicode and UTF-8 for page encryption. We recommend that you use UTF-8 or UTF-16 for your e-mail templates and database connection.
  • Using SSL and TLS encryption on e-mail and LDAP connections
    You can use encrypted connections between Track+ and your e-mail server to prevent passwords from being transmitted clear text across your network.
  • Adding your company logo
    The license conditions permit you to replace the Track+ logo by your own company logo as long as the modified version of Track+ is not distributed outside the company to which this logo belongs.
  • Configuring Track+ with Jenkins
    You can configure Track+ to work together with the Jenkins or Jenkins continuous integration server. This section describes how the procedure of setting Track+ and Jenkins up properly.
  • Configuring Apache Tomcat
    When exposing your Track+ installation to the internet it is usually best to work with an Apache HTTP server or IIS server as a front end to Track+. When you support other applications not running on the same Tomcat server as Track+, but via the same Apache HTTP server, you can still use single sign on without having to install a big SSO solution.
